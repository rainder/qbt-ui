# v1.1 Feature Pack — qBittorrent UI

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Round out the qBittorrent UI with 15 incremental quality-of-life features grouped into 7 focused batches that each subagent can implement in a single sweep.

**Architecture:** Pure additions to existing surfaces. No new pages. New endpoints wired into `src/api/torrents.ts`; new modals into `src/components/Modals/`; new context-menu items into `TorrentTable.tsx`; new TopBar pieces into `TopBar.tsx`.

**Tech Stack:** existing — React 18, TypeScript, Vite, Tailwind v4, TanStack Query, Zustand, React Router.

---

## Conventions

- One commit per batch with a descriptive message starting with `feat:` or `fix:`.
- Use `git -c commit.gpgsign=false commit -m "..."`.
- TDD only for pure-utility additions (e.g., parsers, ring buffers). UI-only changes do not need new tests.
- Reuse `<Select>` (`@/components/ui/Select`), the `inputCls` button classes, and existing modal shell.
- Tailwind v4 utility classes; `text-fg-default`, `bg-canvas-subtle`, `bg-success-emphasis`, etc.
- All new endpoints live in `src/api/torrents.ts` unless `transfer/*` or `log/*`.
- Run `npm run typecheck && npm run test:run && npm run build` before committing.
